How to test TANKS and WOTMOD creations

 

First.. a word about paths.xml
This is a game file used to tell the game where things are located on your disc.

This file is very important. The game will CRASH if its missing or corrupt!!

When you run the game, from either "Test WOTMOD Tank" or "Test RESMODS Tank", Tank Export will make a backup of this file in the World_of_Tanks folder named "paths_backup.xml".
If the paths.xml is corrupt, something broke. The game will CRASH!
You can delete paths.xml and rename paths_backup.xml OR, check the games data in gaming center.
Tank Exporter will try to not mess this file up but it is possible if you start the game from Tank Exporter and then close Tank Exporter before the game has closed. Don't Do That!

What's the difference?
When you click "Test WODMOD Tank", Tank Exporter blocks the game from loading any data from the RES_MODS folder.
and...
When you click "Test RES_MODS Tank", Tank Exporter blocks the game from loading any data from the MOD's folder.

Tank Exporter will minimize and launch World of Tanks.
When you exit the game, Tank Exporter will return a normal window.

How Tank Exporter does this is by modifying the Paths.XML file.
This is why its important not to shut down Tank Exporter if the game was launched from it and still running!
Tank Exporter will be locked while the game is running but there are ways to shut it down.. IE. Task Manager! DONT DO IT!

If the game crashes, you can load the python.log from the Test Tank menu as well.
You can also clear it out of previous startups.
